# Merlin — Medical Record Intelligence

Production website for **Merlin**, a closed-system medical-record intelligence service for
civil-defense and insurance-claims work. A subsidiary of The Guardian Group; a division of
Guardian Group Civil Services.

Creative concept: **“The record, mastered.”** — from thousands of disorganized medical-record
pages to verified, source-grounded legal intelligence.

---

## Stack

- **Next.js 15** (App Router) + **React 19**
- **TypeScript** (strict)
- **Tailwind CSS 3** with centralized design tokens (`tailwind.config.ts` + `globals.css`)
- **next/font** for EB Garamond (display serif), Inter (UI sans), IBM Plex Mono (evidence metadata)
- **next/image** for optimized, responsive imagery (AVIF/WebP)
- Motion is CSS + a small `IntersectionObserver` hook — **no heavy animation library**

| --------------------------------- | ---------------------------------------------------------------------- |

| `DEMO_REQUEST_API_KEY`            | Optional bearer token sent to the endpoint.                            |
| `NEXT_PUBLIC_EXPLAINER_VIDEO_URL` | When set, the explainer mounts a real video (no autoplay audio).       |

### Demo-request form — how delivery works (no faked email)

`POST /api/demo-request` validates the payload server-side, then:

- **If `DEMO_REQUEST_ENDPOINT` is set** → forwards the request there. Success/failure follows the
  downstream service’s response.
- **If it is NOT set** → validates and **logs server-side only**, and the UI clearly states it is a
  development fallback. The site never claims an email was delivered when it was not.

To connect a real provider, implement delivery in `src/app/api/demo-request/route.ts` (the client
submit adapter in `src/lib/submitDemoRequest.ts` never needs to change).

## Content & legal review checklist

These statements are centralized so the client’s legal/compliance team can approve or revise them
in one place (`content/brand.ts` + `content/claims-review.ts`). **Confirm before publishing as
definitive claims:**

| Item | Current treatment |
| ---- | ----------------- |
| **“Developed by physicians. Built for attorneys.”** | **Softened** to “Built by a med-legal team. Made for attorneys.” until `brand.claims.developedByPhysicians.confirmed = true`. |
| **HIPAA posture** | **Withheld.** Add to `trust.verifiedCertifications` only after compliance confirms. |
| **SOC 2** | **Withheld.** Same as above. |
| **Encryption / data-residency** | **Withheld.** Security page describes posture qualitatively only. |
| **Cost vs. physician review** | Published — qualitative only (“a fraction of”). No percentages/figures. |
| **Speed of evaluation** | Published — qualitative only (“in days, not months”). No metrics. |
| **Accuracy** | No “100% accurate”/numeric claim. Framed as grounded in and verifiable against the record; supports, not replaces, professional judgment. |
| **Merlin/Guardian relationship** | Published — exact client wording, centralized in `brand.ts`. |
| **Billing / overbilling language** | Framed as detection/testing (“surfaces”, “warrants a closer look”). No fraud accusation. |

To publish the physician line and certifications once confirmed:

1. `content/brand.ts` → `claims.developedByPhysicians.confirmed = true`.
2. `content/trust.ts` → add confirmed items to `verifiedCertifications`.

## Design decisions (summary)

- **Editorial casebook system**, not a SaaS template: EB Garamond display serif, hairline rules,
  oversized page-marker numerals, monospace “evidence” metadata, and a restrained bronze/gold
  accent over an obsidian / graphite / ivory / paper base. Bronze and gold are used sparingly; no
  blue/purple/cyan “AI” gradients.
- **Positioned as a closed, curated record-intelligence service** — not “just another AI chatbot.”
  Copy leads with command, verification, and security; AI is the enabling technology, not the pitch.
- **Recurring source-citation motif** (`EvidenceCitation`) reinforces that every answer is grounded
  in a real page of the record. All record data shown is obviously fictional — **no real PHI**.
- **Signature moments:** the sealed-case hero visual (`RecordStackVisual`) and the
  “Who reviewed the reviewers? → Merlin did.” provocation band (revealed on scroll).
- **Accessibility & motion:** semantic HTML, skip link, keyboard-accessible accordion/menu,
  visible focus, WCAG-AA-minded contrast, and full `prefers-reduced-motion` support — all essential
  content is understandable without animation or JavaScript.

## Accessibility & performance notes

- Semantic landmarks, single `h1` per page, labelled form fields with inline errors, ARIA on the
  accordion and mobile dialog, and a visible skip-to-content link.
- Responsive from 360px to 1440px+; fluid type via `clamp()`; tables/indexes reflow; no horizontal
  overflow; 16:9 video ratio preserved; large tap targets on mobile.
- Optimized images via `next/image`, lazy-loaded non-critical media, minimal client JS.
